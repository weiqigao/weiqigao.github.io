Title: Jenkins 2.0 Overview
Presented By: Chris Vogel
About the Presentation:

Jenkins 2.0 was recently released with a focus on the use case of a continuous delivery pipeline. In order to make the move from continuous integration to continuous delivery, teams need a flexible way to model, orchestrate and visualize their entire delivery pipeline. This is why they added the pipeline capability to Jenkins 2.0. This new capability comes in the form of "Pipeline as Code," using a Groovy-based DSL that can be version-controlled with the rest of a project's source code. In this talk, we will use a Docker containerized web application as an example to look at the Jenkins DSL. We will also look at the new user interface, "Ocean Blue," and its tie-in to pipelines.
About the Presenter:

Chris Vogel is a Technical Architect at Edward Jones, where he has worked for the last twenty years. He has worked with Java since 1997. The majority of his career has been spent helping set the application development direction and supporting teams by defining application architecture, supporting developer tools and mentoring.
