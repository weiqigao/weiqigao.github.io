Overview of Ant

By Mark Volkmann, Object Computing, Inc.

About Ant

Ant is a build tool like GNU make. It's an open source product from the Apache Jakarta project that is implemented in Java. Many open source products, such as Tomcat and JDOM, use it as their primary mechanism for compiling, creating JAR/WAR files, generating javadoc and installing software.

Why use Ant instead of GNU make?

  • Ant is more portable. It only requires a Java VM (1.1 or higher). GNU make relies on OS specific commands to carry out tasks. GNU make can be used under Windows using Cygwin (a UNIX emulator) but that's a big install (~37 meg.)!
  • Ant targets are described in XML. GNU make has a cryptic syntax and relies proper use of tabs that is easy to get wrong since you can't see them.
  • Ant is best suited for Java-specific tasks. It is faster than GNU make since all tasks are run from a single JVM.
  • Ant is easier than GNU make for Java-specific tasks such as generating javadoc, building JAR/WAR files and working with EJB.
