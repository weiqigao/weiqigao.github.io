September, 2000

Servlet Framework

By Eric Burke

About the Presentation

The Java Servlet API is a class library for developing portable web applications in Java. Although Servlets make it much easier to create dynamic web pages, it is up to the developer to figure out how to assemble these pages into a complete web application. Typical textbook examples provide a single Servlet which intercepts requests from a browser, performs some business logic, and then generates an HTML page as a response.

Unfortunately, this approach does not scale well. A fundamental problem is that the Servlet is doing too much. Intercepting client requests, validation logic, and page rendering should be broken out into separate classes.

Eric Burke will present an open source application framework which builds on the Java Servlet API. This framework defines an object-oriented approach to building web applications where page rendering and request processing are clearly separated into different classes. This makes it possible to develop page rendering classes without specific knowledge of other screens or overall application flow.

About Eric Burke

Eric Burke is a Senior Software Engineer with Object Computing, Inc. and has been teaching Java for over three years. He has authored several Java courses, including "Java Servlet Programming" and "Advanced Servlet and JSP Programming".
