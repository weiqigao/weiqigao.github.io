Title: Model-Oriented Architectures /
Integrating Information using Models and Metadata
Presented By: Randall Hauch and Bradley Wright, MetaMatrix
About the Presentation:

Model-oriented architecture (MOA) is a new term for a powerful concept that has been gaining popularity for several years: using models and metadata to drive in how systems work and operate. Models and modeling technologies (e.g., UML) have been used in the development process for many years. However, a model-oriented system can actually consume models at runtime and uses those models to determine how the system should behave. This presentation will discuss the benefits of MOAs, some of the standards involved (such as MOF, XMI, CWM, XSD, etc.), some available technologies (including Eclipse and EMF), and how development changes for model-oriented systems. The presentation will describe how the MetaMatrix products enable customers to model their disparate physical data sources and the information needed by consumers, and then to simply deploy those models as a "virtual database" to provide unified and real-time integrated information through JDBC, ODBC and SOAP.

Competition and limited resources are requiring organizations to provide more quickly information that is more accurate to more consumers. Providing all information consumers with accurate, complete, and integrated views of all the information needed is extremely challenging when that information is actually managed in many data sources with different schemas and persistence technologies. Custom code to perform this integration is often not reusable, error-prone, inefficient and subject to changes in the data sources. Warehouses are expensive, time-consuming and make "real-time" access difficult. However, "virtual databases" can expose the actual data in the desired structure to each consuming application, yet abstracts away from the application the details of how the underlying data sources are accessed and how the information is integrated. This presentation will discuss how MetaMatrix defines these "virtual databases" using models and metadata, and how those models are deployed to an integration server to which clients can connect via JDBC, ODBC, and SOAP.
