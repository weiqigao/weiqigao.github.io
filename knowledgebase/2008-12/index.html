Title: Test NG
Presented By: C. Gordon Huffman, Daugherty
About the Presentation:

TestNG, invented by Google's Cèdric Beust, is a JUnit-inspired automated testing Java framework which aims to solve the limitations of JUnit.

I'll explain the differences in best practices for automated testing of a larger (integration) vs. a smaller (unit) amount of code (whether you use TestNG or JUnit).

I'll explain how to effectively use TestNG (and JUnit) features to achieve best practices in both of these arenas.

If you're interested in whether TestNG lives up to its acronym ("Next Generation"), or if you want to know how to take JUnit to the next level, come hear how you can construct better tests with either of these tools!
About the Presenters:

C. Gordon Huffman has been in software development for over 28 years, following a brief but enjoyable career as a university and college mathematics instructor. He has been in the Java world since 1997. Gordon is currently an application architect and developer with Daugherty Business Solutions.
