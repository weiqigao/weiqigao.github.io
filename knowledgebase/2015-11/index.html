Title: Contexts and Dependency Injection (CDI)
Presented By: Michael Remijan
About the Presentation:

The purpose of this talk is to introduce Contexts and Dependency Injection (CDI) and to also present some of the new features coming with the early draft release of CDI 2.0 (JSR 365). CDI is the Java standard for dependency injection. It is a critical part of Java EE, but it is a stand-alone standard which can also be used within Java SE environments. The talk will start with an overview of the CDI specification and cover basics: Injection, Producers, Qualifiers, and Observers. After that, will be a discussion of some of the new features planed for CDI 2.0 which includes priority observers, asynchronous observers, and the new standard for bootstrapping CDI in a Java SE environment. For this talk, all the examples will use Weld, which is the reference implementation of the CDI specification.
About the Presenter:

Michael Remijan is a Systems Architect at the Federal Reserve Bank. He is a co-author of the EJB In Action 2nd edition book, a participant of JSR 365 (Contexts and Dependency Injection for Java 2.0), and nominated for the expert group of JSR 368 (Java Message Service JMS - 2.1). He has been working with Java EE technology almost since it's beginning, starting in 1999. He has used just about every part of the Java EE specification in one way or another over the years. He is an evangelist of Java EE technology, and likes to develop to standards as much as possible.
